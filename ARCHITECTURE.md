# VidJobs - Архитектура и документация проекта

## 📱 Обзор приложения

VidJobs - это цифровая платформа для быстрого размещения и выполнения краткосрочных задач с видео-описанием. Приложение соединяет заказчиков и исполнителей с помощью инновационной системы, где заказы описываются через видео/голос, а AI автоматически генерирует техническое задание.

## 📊 Модели данных

```python
- owner (FK User)           # Создатель заказа

- category (Choice)         # Категория работ
- budget (Integer)          # Бюджет в ₸
- status (Choice)           # Статус (черновик, опубликован, назначен, в процессе, завершен)
- priority (Choice)         # Приоритет
- video_file (File)         # Видео-описание
- voice_file (File)         # Голосовое описание
- ai_generated_description  # AI сгенерированное ТЗ
- location_latitude/longitude # Геолокация
- deadline (DateTime)       # Срок выполнения
```

## 🔐 Система авторизации

### SMS Верификация
- Пользователь вводит номер телефона
- Отправляется SMS код подтверждения
- После ввода кода пользователь верифицирован
- Сохраняется в `Profile.phone_verified`

### Верификация для исполнителей
- Загрузка удостоверения личности
- Проверка документов администратором
- Установка `verification_status = 'verified'`
- Только верифицированные исполнители могут подавать предложения

### Token-based Authentication
- JWT токены для API
- Refresh токены для обновления
- Secure HTTP-only cookies для хранения

## 💰 Финансовая модель

### Комиссия платформы
- **5% от суммы заказа** - стандартная комиссия
- **Первые 3 месяца free** - для привлечения пользователей
- Комиссия автоматически рассчитывается при создании заказа

### Расход платежа
```
Бюджет заказа: 100,000₸
├─ Комиссия (5%): 5,000₸ → Кошелек платформы
└─ К выплате исполнителю: 95,000₸
```

### Система кошелька
- Каждый пользователь имеет `wallet_balance`
- Деньги идут в кошелек при завершении заказа
- Можно выводить деньги (вывод на карту/счет)
- История всех операций в `PaymentTransaction`

## ⭐ Система рейтинга

### Критерии оценки
1. **Качество выполнения** (1-5 звезд)
   - Соответствие требованиям
   - Профессионализм работы

2. **Соблюдение сроков** (1-5 звезд)
   - Вовремя ли завершена работа
   - Пунктуальность

3. **Коммуникация** (1-5 звезд)
   - Отзывчивость
   - Контакт с заказчиком

### Влияние рейтинга
- **Высокий рейтинг** (4.5+):
  - Первым появляется в поиске
  - Больше предложений от заказчиков
  - Возможность поднять цены

- **Низкий рейтинг** (<2.0):
  - Может быть блокирован
  - Требуется повышение рейтинга
  - Меньше видимости

### Расчет рейтинга
```
Average Rating = (Quality + Timeline + Communication) / 3
Total Reviews = Count(all ratings)
```

## 🎥 Обработка видео/голоса (AI)

### Процесс
1. **Загрузка** - пользователь загружает видео (макс 30 сек) или голос
2. **Обработка** - AI преобразует в текст
3. **Генерация ТЗ** - AI создает профессиональное ТЗ
4. **Подтверждение** - пользователь подтверждает или редактирует
5. **Сохранение** - ТЗ сохраняется в БД

### AI Интеграция (Future)
- Используется для:
  - Speech-to-text (голос в текст)
  - Text-to-description (текст в профессиональное ТЗ)
  - Image analysis (анализ фотографий)

### Уверенность AI
- Сохраняется `confidence_score` (0.0-1.0)
- Если уверенность < 0.7 - требуется подтверждение пользователя
- Если < 0.5 - пользователь должен отколить описание вручную

## 📍 Геолокационный поиск

### Функционал
- **Заказчик** указывает место работ
- **Исполнитель** указывает радиус обслуживания (по умолчанию 50 км)
- **Система** автоматически фильтрует заказы в радиусе

### Поиск по локации
```sql
SELECT jobs 
WHERE distance(job.location, executor.geolocation) 
      <= executor.service_radius
ORDER BY distance ASC
```

### Приватность
- Точные координаты скрыты до согласия
- Показывается только радиус и адрес района
- Полные координаты раскрываются после принятия заказа
